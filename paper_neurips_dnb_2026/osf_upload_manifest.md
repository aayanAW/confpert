## OSF upload steps (user-side, ~15 minutes)

1. Visit https://osf.io/ and sign in (or create an account using your independent-researcher identity).
2. Create a new project: "ConfPert Phase 2 Pre-Registration" (or similar).
3. Mark the project as **Pre-Registration** type (not generic Project).
4. Upload the four files listed above.
5. Mark the registration as **Open-Ended Registration** (allows updates if pre-reg v2.1 is needed later; locks the v2.0 version permanently).
6. Copy the resulting OSF DOI (format: `10.17605/OSF.IO/<8-char-code>`).
7. Back in this repo, run:
   ```bash
   cd /Users/aayanalwani/FLM4S/confpert
   # Update the lock.osf_doi field in the YAML
   python -c "
   import yaml
   p = 'paper_neurips_dnb_2026/preregistration_v2.yaml'
   with open(p) as fh: s = yaml.safe_load(fh)
   s['lock']['osf_doi'] = '<PASTE_OSF_DOI_HERE>'
   with open(p, 'w') as fh: yaml.safe_dump(s, fh, sort_keys=False, default_flow_style=False)
   "
   # Re-emit hashes to lock in the OSF DOI
   python -m confpert.cli prereg emit-hashes --prereg paper_neurips_dnb_2026/preregistration_v2.yaml
   # Regenerate the .sha256 file
   cd paper_neurips_dnb_2026
   shasum -a 256 preregistration_v2.md preregistration_v2.yaml > preregistration_v2.sha256
   # Re-verify
   cd ..
   python -m confpert.cli prereg verify --prereg paper_neurips_dnb_2026/preregistration_v2.yaml --dry-run
   ```

## Why an OSF mirror

- **Permanent timestamp:** OSF assigns an immutable DOI + timestamp. Even if the local git history is rebased, the OSF registration cannot be changed.
- **Scoop defense:** the OSF DOI predates the Phase 2B-2F work. Any later paper that overlaps with K2 v2's H2/H2b/H3 framing can be unambiguously dated against this registration.
- **Reviewer-defensible:** D&B reviewers expect external pre-reg evidence beyond git. OSF is the field-standard.
- **Per Hofman 2023:** OSF is one of the explicit recommended platforms for ML pre-registration.

## What NOT to upload to OSF yet

- `baselines/results.json` — uploaded only after Phase 2 results merged, with appropriate license disclaimer
- Phase 2 paper draft — wait until ICLR / NeurIPS D&B submission complete
- Predictor weights or fine-tuned models — too large; reference via separate Modal/HF artifact links

## Verification at OSF lock-time

After OSF upload, the YAML's `lock.osf_doi` becomes the canonical pointer to the registration. The `confpert prereg verify` output JSON will include this DOI in every verification report, so downstream analysts can cross-check against the public registration.
